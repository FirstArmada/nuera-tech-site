## 2026-06-13 - [CRITICAL] Fix rate-limit bypass via IP spoofing
**Vulnerability:** The Express application in `cloud/chat-proxy/server.js` was parsing the `X-Forwarded-For` header manually (`req.headers['x-forwarded-for']`) to determine the client IP for rate limiting. Without configuring Express to properly trust the proxy, malicious clients could trivially spoof this header, allowing them to bypass rate limits.
**Learning:** When deploying Express applications behind reverse proxies like Cloud Run, manual parsing of `X-Forwarded-For` is insecure. Express cannot verify which proxy appended the IP address, allowing an attacker to inject arbitrary IPs in the header and evade IP-based blocking.
**Prevention:** Always configure `app.set('trust proxy', 1)` (or the appropriate depth) in Express when deployed behind a known reverse proxy (e.g., Cloud Run). This allows Express to securely handle `req.ip` by safely parsing proxy headers, mitigating IP spoofing attacks. Use `req.ip` instead of manually reading headers.

## 2024-05-24 - Rate Limit Bypass via IP Spoofing
**Vulnerability:** The chat-proxy endpoint relied on `String(req.headers['x-forwarded-for']).split(',')[0]` to determine the client IP. This allowed attackers to bypass the 20 requests/min rate limit by supplying a spoofed `X-Forwarded-For` header containing a fake IP in the first position, effectively giving them an infinite number of independent token buckets.
**Learning:** `X-Forwarded-For` is a comma-separated list where the leftmost value is easily spoofed by the original client, and subsequent values are appended by proxies in the chain. Extracting the first value blindly is unsafe.
**Prevention:** Always configure the framework's native proxy trust mechanism (e.g., Express `app.set('trust proxy', 1)`) when deployed behind a known proxy topology (like Cloud Run). This allows the framework to reliably extract the true client IP appended by the trusted proxy, securely dropping spoofed prefixes, and exposing it as `req.ip`.
